Laboratorium 1

Organizacja kodu źródłowego w Javie

  1. Pliki źródłowe programu w Java maja rozszerzenie *.java
  2. Pliki uruchamialne maja rozszerzenie *.class
  3. Każda klasa publiczna musi znajdować się w osobnym pliku java.
  4. Klasy mogą być grupowane w tzw. pakiety. Pakiet zazwyczaj zawiera w sobie klasy implementujące różne funkcjonalności z tego samego lub podobnego obszaru (np. pakiet java.io zawiera klasy odpowiedzialne za wejście/wyjście w języku Java). Pakiet jest katalogiem w systemie plików.

Kompilacja/Uruchamianie

  1. Do kompilacji używamy polecenia javac Przykładowo chcąc skompilować program zawierający 2 klasy HelloWorld.java znajdującej się w pakiecie myPackage oraz JIn.java znajdującej sie w pakiecie javaIn wywołujemy następujące polecenie: 
     javac myPackage/HelloWorld.java javaIn/JIn.java
  2. Aby uruchomić program HelloWorld wywołujemy polecenie 
     java myPackage/HelloWorld

    Nie podajemy rozszerzenia *.class pliku

Struktura programu w Javie

  1. W Java wszystko jest obiektem, dlatego metoda main (konieczna aby dało się uruchomić program) także musi być metodą jakiejś klasy. Zazwyczaj istnieje jedna klasa służąca jedynie do uruchamiania logiki aplikacji, ale nie implementująca jej, która zawiera metodę main.
  2. Metoda main musi być zdefiniowana jak pokazano poniżej; musi być metodą publiczną, statyczną, zwracającą void oraz przyjmującą tablicę napisów jako parametr.
  3. Program zazwyczaj składa się z kilku plików, które mogą znajdować się w różnych pakietach. Przynależność klasy do danego pakietu należy zaznaczyć w kodzie źródłowym.
  4. Program może korzystać z klas znajdujących się w innych pakietach. Należy wówczas zaimportować te klasy, lub całe pakiety; służy do tego dyrektywa import (analogiczna do #include z C++).
  5. Każdą klasę powinno umieszczać się w osobnym pliku o tej samej nazwie co dana klasa.

Poniżej - klasa do wczytywania napisu, znajdująca się w pakiecie javaIn. Korzysta ona z klas wchodzących w skład pakietu java.io. 

package javaIn;
 
import java.io.*;
 
public class JIn {
    public static String getString() {
	  String text = null;
	  try{
		InputStreamReader rd = new InputStreamReader(System.in);
		BufferedReader bfr = new BufferedReader(rd);
 
		text = bfr.readLine();
	  }catch(IOException e){e.printStackTrace();}
      return text;	  
    }
}

Pamięć

W Javie zmienne są referencjami, a nie bezpośrednimi uchwytami do wartości jak w przypadku C++. Z tego powodu wszystkie zmienne (poza typami prostymi i przypadkami automatycznego rzutowania) są tworzone za pomocą operatora new: 

Integer zmienna1 = new Integer(12); // bezpośrednie wywołanie konstruktora
Integer zmienna2 = 12; // automatyczne rzutowanie

W Javie nie ma operatora delete zwalnianiem pamięci zajmuje się Garbage collector, który automatycznie usuwa obiekty, które nie są już nigdzie używane.

Typy Proste

W języku Java istnieją następujące typy proste, nie będące obiektami (maja jednak swoje obiektowe odpowiedniki):

  • boolean - true or false
  • byte - 8-bit integer
  • short - 16-bit integer
  • int - 32-bit integer
  • long - 64-bit integer
  • float - 32-bit IEEE real number
  • double - 64-bit IEEE real number
  • char - Unicode character

Zmienne typów prostych przekazywane są do funkcji przez wartość.

W odniesieniu do obiektów, każda zmienna jest jedynie referencją do obiektu. Przekazywanie obiektu do funkcji jest zatem przekazywaniem referencji (analogicznie w przypadku operatorów = oraz ==).

Instrukcje sterujące

Standardowe instrukcje sterujące:

  • if oraz if … else
  • switch
  • while oraz do … while
  • for
  • break
  • continue
  • return

Słowo kluczowe "static"

  1. Może być stosowane w odniesieniu do metod i pól klasy. Zmienne i metody statyczne nie są związane z obiektami, ale z klasami. Statyczne pole lub metoda będą współdzielone przez wszystkei obiekty danej klasy.
  2. Do metod i pól statycznych można odwoływać się nie tworząc obiektu danej klasy: 
    public class A{
  public static void sayHello(){
    System.out.println("Hello world!");
  }
}
 
...
 
public class B{
  public static void main(String[] argv){
    A.sayHello();
    /* Nie trzeba tworzyć obiektu klasy A */
  }
}
  3. Wewnątrz statycznej metody nie można wywoływać niestatycznych metod danej klasy.
  4. Statyczne metody nie mogą być nadpisywane w klasach pochodnych

JavaDoc

Narzędzie JavaDoc pozwala na automatyczne tworzenie dokumentacji identycznej z dostępną na stronie Java API. Wszelkie informacje dla narzędzia JavaDoc zamieszczane są w komentarzach. 

package math;
/**
 * Klasa licząca średnia ocen podanych jako parametr
 * 
 */
public class Average {
    /**
     * Metoda oblicza średnia ocen
     * @param grades lista ocen jednostkowych
     * @return średnia ocen podanych jako parametr
     */
    public double calculate(double[] grades){
        double sum = 0;
        for(int i = 0; i < grades.length; i++){
            sum += grades[i];
        }
        return sum/grades.length;
    }
}

Generowanie dokumentacji wykonywane jest po wywołaniu polecenia 

javadoc -d "katalog docelowy" -sourcepath "katalog ze źródłem" pakiet1 pakiet2 ... pakietn
